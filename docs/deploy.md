# Deploying the Moneyball Backend

**Host:** [Render](https://render.com) free tier (Web Service, Docker runtime).
Free tier auto-sleeps after 15 min idle — first request wakes it in ~30 s.

---

## 1. Create the service

1. Go to **Render Dashboard → New → Web Service**.
2. Connect the `anna-stolbovskaja/moneyball` repo.
3. Render auto-detects `render.yaml` — confirm the blueprint.
4. The Dockerfile at `apps/backend/Dockerfile` builds the image.

## 2. Set environment variables

In the Render dashboard, set the following (see `apps/backend/.env.example` for full list):

| Variable | Required | Notes |
|---|---|---|
| `CORS_ORIGINS` | ✅ | Your Walrus Sites URL, e.g. `https://abc123.wal.app` |
| `JWT_SECRET` | ✅ | Auto-generated by Render blueprint |
| `AUTH_DOMAIN` | ✅ | Same as your Walrus Sites domain |
| `AUTH_URI` | ✅ | Full URL of your frontend |
| `ADMIN_ALLOWLIST` | ✅ | Your Sui address(es), lowercase, comma-separated |
| `MEMWAL_KEY` | ✅ | From [memory.walrus.xyz](https://memory.walrus.xyz) |
| `MEMWAL_ACCOUNT_ID` | ✅ | From MemWal dashboard |
| `FOOTBALL_DATA_TOKEN` | optional | Only if `MATCH_SOURCE=football-data` |

All other vars have safe defaults (see `render.yaml`).

## 3. Deploy & verify

```bash
# After deploy completes, check health:
curl https://moneyball-backend.onrender.com/health
# Expected: {"ok":true,"ts":"2026-06-12T..."}

# Test Socket.io handshake:
curl "https://moneyball-backend.onrender.com/socket.io/?EIO=4&transport=polling"
# Expected: 0{...} (Engine.IO open packet)
```

## 4. Point frontend to backend

In your frontend deploy (Walrus Sites or local dev), set:

```
VITE_BACKEND_URL=https://moneyball-backend.onrender.com
```

## 5. CORS checklist

The backend reads `CORS_ORIGINS` as a comma-separated list. You must include:

- Your Walrus Sites URL: `https://<site-id>.wal.app`
- (Optional) `http://localhost:5173` for local dev

Example: `CORS_ORIGINS=https://abc123.wal.app,http://localhost:5173`

Socket.io also respects these origins via the same CORS config in `src/index.ts`.

## Local Docker build

```bash
# From repo root:
docker build -f apps/backend/Dockerfile -t moneyball-backend .

# Run locally:
docker run -p 3001:3001 --env-file apps/backend/.env moneyball-backend

# Verify:
curl http://localhost:3001/health
```
